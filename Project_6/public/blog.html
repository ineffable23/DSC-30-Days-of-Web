30 Days of Web Development

Organized by a total of 50+ DSC Leads over the world.

  • Github
  • HTML
  • CSS

30 Days of Web Development provides us with an opportunity to kick start our journey in web development and get hands-on experience with HTML, CSS, JS, Bootstrap, and some basics of back-end.


The main objective behind this program was to provide the best content possible at no cost, with sessions from awesome speakers and skilled mentors who were there to help us throughout the process.


Along the way, we worked with HTML, CSS, JS, Bootstrap, Node JS basics with the help of beginner-friendly content, we have designed. We also had to work on some quizzes based on what we learned along with some projects which helped us get better with the basics each day.


Github

We started up with the basics of Git ans Github.

HTML

Day 1

We started up with the basics of HTML. We went through formatting in HTML, working of a link , various tags, and many more.

Day 2

We worked with tables and lists in HTML. Tables are an essential part of HTML as we can make many different patterns out of it.

Day 3

We focused on debuging our HTML code effectively and work on creating forms.

Day 4

We worked with Media in HTML. Media is helpful when we want to make our website or webpage look attractive by providing some interactivity to the user on the go.

Day 5 (Optional)

We worked with APIs in HTML. API is one of those things which we usually use many times in our project to make our code lighter and faster.

CSS

Day 1

We started up with the basics of CSS. We got to know how colors, backgrounds, borders & margins work

.

Day 2

We worked with padding, height and widths of an element.

Day 3

We'll focus on texts, icons, typography, forms, grids, and many more.

Day 4

We started with some important concepts in CSS. We got to know how we can round the corners of elements, add borders, do transformations, and many more.

Day 5 (Optional)

We worked with some more advanced things in CSS like transformation, media queries, responsive grids, and so on. .

Bootstrap

Day 1

We started up with understanding the difference between Bootstrap version 3 and Bootstrap version 4 . and then understood the Installation steps and other feautres of Bootstrap.

Day 2

We looked into more advanced concepts of Bootstrap such as Jumbotron, Alerts, Collapse, Tooltip, Modals, Scrollspy, Toast, Pagination, Spinners, Utilities and some Bootstrap Examples.

JavaScript

Day 1

We started with what browser-side scripting is, what JS is, how we can work with js, and a hello world program in JS.

Day 2

We started with where we left yesterday. We saw how we can initialize things in Js, data types, some basic arithmetic operations, and a brief about functions.

Day 3

We worked with arrays, classes, and objects in JavaScript. We also saw how inheritance works in JS.

Day 4

Arrow functions are one of those things which have helped to make code in javascript compact. We learnt how we can implement them in our code. Apart from this we also worked with Spread and Rest Operators and had a quick look at the Document Object Model(DOM).

Day 5

We started with how we candestructure objects and then with Document object manipulations in JS.

Day 6

We worked on the Document object model and interacted with the web pages a bit more.

Firebase

Day 1

We gained some knowledge about how the internet works and how the client-server model works.

Day 2

We had a brief introduction on what is Firebase and how it is useful. We made our account on firebase, got familiar with the dashboard, created a project and integrated it with our website. We also saw how we can host our website using firebase.

Day 3

We got to know about Cloud Firestore and Realtime Database. We also created a project, performed CRUD operation and integrated it with our website.

Day 4

We got to know how we can store and render images to firebase and authentication to our website.

Day 5

We learnt how to make real-time chat website where everyone can chat with each other using firebase as our backend and database.
